# 🚀 Nano Banana Pro 部署指引 (雨云 + 宝塔面板)

本教程专为初学者设计，旨在帮助您将本项目顺利部署到雨云服务器上。

## 📋 准备工作

在开始之前，请确保您的宝塔面板已安装：
1.  **Nginx** (用于托管前端和设置转发)
2.  **Node.js 版本管理器** (在软件商店搜索安装)

---

## 🛠️ 第一步：本地项目打包

在您的本地电脑（VS Code）上：

1.  打开内部终端。
2.  输入命令：`npm run build`
3.  等待完成后，项目根目录下会出现一个 **`dist`** 文件夹。这个文件夹里包含了网页的所有静态文件。

---

## 📂 第二步：上传文件到服务器

1.  登录宝塔面板，进入 **“文件”** 菜单。
2.  导航到网站存放目录（通常是 `/www/wwwroot/`），新建一个文件夹，例如 `nano-banana`。
3.  进入该文件夹，点击“上传”，将以下内容上传：
    *   **本地 `dist` 文件夹内的所有文件** (注意：是 dist 里面的内容，直接拖进去)。
    *   **本地根目录的 `server.cjs`** (这是后端代理服务)。
    *   **本地根目录的 `package.json`** (用于安装运行环境)。

---

## ⚙️ 第三步：启动后端服务 (Node.js)

1.  进入宝塔 **“网站” -> “Node.js 项目”**。
2.  点击 **“添加 Node 项目”**：
    *   **项目执行文件**：选择您上传的 `server.cjs`。
    *   **项目名称**：随便填，比如 `banana-api`。
    *   **项目端口**：填 `3323`（这是我们设置的后端端口）。
    *   **运行用户**：www。
3.  点击确定，系统会自动扫描并尝试安装依赖（express, axios, cors）。
4.  确认状态显示为绿色 **“运行中”**。

---

## 🌐 第四步：部署前端网站

1.  进入宝塔 **“网站” -> “HTML 项目”** (或 PHP 项目，效果一样)。
2.  点击 **“添加站点”**：
    *   **域名**：填写您的服务器 IP 或您买的域名。
    *   **根目录**：选择刚才上传文件的文件夹（即 `/www/wwwroot/nano-banana`）。
3.  提交后，您现在访问 IP 应该能看到网页了，但由于还没连接后端，生图功能暂时不可用。

---

## 🔄 第五步：设置反向代理 (关键步骤)

我们需要让网页发送到 `/api` 的请求自动转发到后端的 `3323` 端口。

1.  在刚才创建的站点列表里，点击右侧的 **“设置”**。
2.  在左侧菜单找到 **“反向代理”**。
3.  点击 **“添加反向代理”**：
    *   **代理名称**：填 `api`。
    *   **目标 URL**：填 `http://127.0.0.1:3323`。
    *   **发送域名**：`$host`。
4.  保存即可。

---

## 🛡️ 第六步：安全组放行

1.  在宝塔面板的 **“安全”** 菜单中，放行 `80` 和 `3322`（如果您想通过 3322 端口直接访问网站）。
2.  在 **雨云控制台** 的“弹性云服务器” -> “防火墙”里，也要放行对应的端口。

---

## 💡 常见问题排查

*   **页面刷新 404**：在站点设置 -> 伪静态中，粘贴以下代码：
    ```nginx
    location / {
        try_files $uri $uri/ /index.html;
    }
    ```
*   **连接服务器失败**：检查 Node.js 项目是否在正常运行。
*   **API 错误**：请确保您在网站的【设置】中正确输入了 API Key。
